# 前端服务运行错误项目导致界面显示不正确

**ID**: KB-P-004
**分类**: errors
**创建时间**: 2026-01-29
**最后使用**: 2026-01-29
**使用次数**: 0

---

## 问题描述

前端服务启动后，浏览器访问显示的是错误的项目界面。具体表现为：
- 端口被占用但运行的是其他项目
- 界面显示的系统名称不正确
- 菜单和功能与预期不符

**常见场景**:
- 同一台机器上有多个前端项目
- 之前启动的项目进程未正确关闭
- 端口被其他项目占用
- 在错误的目录下启动了前端服务

---

## 解决方案

### 方法1: 检查并重启正确的前端服务（推荐）

**步骤**:

1. **检查端口占用情况**
   ```bash
   # Windows
   netstat -ano | findstr "5173"
   netstat -ano | findstr "3000"
   
   # Linux/Mac
   lsof -i :5173
   lsof -i :3000
   ```

2. **停止错误的进程**
   ```bash
   # Windows
   taskkill //F //PID <进程ID>
   
   # Linux/Mac
   kill -9 <进程ID>
   ```

3. **确认当前目录**
   ```bash
   pwd  # 确保在正确的项目目录
   ls frontend/  # 确认 frontend 目录存在
   ```

4. **重新启动前端服务**
   ```bash
   cd frontend
   npm run dev
   ```

5. **验证服务**
   - 检查终端输出的端口号
   - 访问浏览器确认界面正确
   - 检查系统名称和菜单是否正确

**成功率**: 95%

**注意事项**:
- Windows 下 taskkill 命令需要使用双斜杠 `//F` 而不是 `/F`
- 确保在正确的项目目录下执行命令
- 如果端口仍被占用，可能需要重启电脑

---

### 方法2: 使用不同的端口

如果默认端口被占用且无法释放：

```bash
# 修改 vite.config.js 或在命令行指定端口
npm run dev -- --port 3001
```
