# Phase 2 测试总结简报

**日期**: 2026-02-02
**实验**: P7.1 双文本策略（数据驱动属性词发现）
**结论**: ⚠️ 效果不佳，需要调整策略

---

## 📊 一句话总结

**双文本策略单独使用只带来3.1%的改善（1,392 → 1,349个簇），远未达到目标（300-500个簇）。**

---

## 🎯 核心数据

### 聚类结果对比

```
Phase 1 基线:  1,392 个簇
Phase 2 结果:  1,349 个簇
改善幅度:      -43 (-3.1%)
目标:          300-500 个簇
达成情况:      ❌ 未达成
```

### 簇分布对比

| 类型 | Phase 1 | Phase 2 | 变化 |
|------|---------|---------|------|
| 主要簇 (≥10) | 215 (15.4%) | 211 (15.6%) | -4 |
| 次级簇 (5-9) | 458 (32.9%) | 461 (34.2%) | +3 |
| 微型簇 (3-4) | 719 (51.6%) | 677 (50.2%) | -42 |
| 噪音点 | 4,473 (28.3%) | 4,631 (29.3%) | +158 |

### 发现的属性词

**5个属性词** (分散度阈值: 0.3):
1. template (0.522)
2. digital (0.463)
3. download (0.408)
4. bundle (0.361)
5. canva (0.310)

---

## ❓ 为什么效果这么小？

### 1. 属性词太少 (5个)

**问题**: 阈值0.3太高，漏掉了很多属性词

**证据**:
- templates (0.289) - 刚好低于阈值
- editable (0.249) - 明显的属性词
- pdf (0.213) - 格式词

**影响**: 只去除5个词，对整体聚类影响很小

---

### 2. Stage 2/3 制造微型簇 (677个)

**问题**: 结构性问题，预处理无法解决

**机制**:
```
Stage 1: "Pink Retro Shopify Theme" → 噪音点
Stage 2: 创建新簇 (3个商品)
         ↓
即使去除 "template"：
Stage 2: 仍然创建新簇 "Pink Retro Shopify" (3个商品)
```

**影响**: 677个微型簇仍然存在

---

### 3. 文本变短增加噪音 (+158)

**问题**: 去除属性词后，文本信息减少

**示例**:
```
原文本: "Canva Template 1" → 有足够信息聚类
新文本: "1" → 信息太少，变成噪音
```

**影响**: 噪音点从4,473增加到4,631

---

## 💡 关键洞察

### ✅ 成功的地方

1. **属性词发现算法有效**
   - 识别出的5个词确实是高分散度词
   - 数据驱动方法比预定义词表更灵活

2. **技术实现正确**
   - 双文本策略实现完整
   - 测试流程顺利完成

### ❌ 失败的地方

1. **单一优化效果有限**
   - 只优化预处理，无法解决结构性问题
   - 需要组合多个优化策略

2. **参数设置不当**
   - 阈值0.3太高
   - 应该降低到0.2以识别更多属性词

3. **根本问题未解决**
   - Stage 2/3 制造微型簇的问题仍然存在
   - 必须重新设计 Stage 2/3

---

## 🎯 下一步行动

### 方案A: 调整参数 (快速测试) ⚡

**做什么**: 降低阈值 0.3 → 0.2

**预期**:
- 识别出15-20个属性词
- 簇数量: 1,349 → 1,000-1,200
- 改善幅度: 3.1% → 10-15%

**时间**: 30分钟

**优点**: 快速验证假设
**缺点**: 仍然无法达到目标

---

### 方案B: Stage 2/3 重新设计 (推荐) 🔴

**做什么**:
- Stage 2: 归并到最近主题簇（不创建新簇）
- Stage 3: 质量门控（只保留可命名簇）

**预期**:
- 消除677个微型簇
- 簇数量: 1,349 → 400-600
- 改善幅度: 3.1% → 50-60%

**时间**: 2-3小时

**优点**: 解决根本问题
**缺点**: 需要重新设计

---

### 方案C: 组合策略 (最佳) ⭐

**做什么**: 方案A + 方案B + 参数优化

**预期**:
- 簇数量: 1,349 → 300-500 ✅
- 微型簇: 677 → <100 ✅
- 可命名簇比例: >80% ✅

**时间**: 半天

**优点**: 系统性解决，达到所有目标
**缺点**: 需要较多时间
